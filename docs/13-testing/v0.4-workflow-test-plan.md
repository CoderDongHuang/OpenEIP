# v0.4 Workflow Test Plan

> Issue: #70 | Coverage gate: >= 80%

| Area | Required evidence |
|---|---|
| Definition | draft revision conflict, strict graph fields, limits, reachability, cycle, port, publish digest and restore |
| Execution | persisted ordered transitions, deterministic fan-out/join, approval/delay suspension, cancellation and restart continuation |
| Trigger | manual/webhook/UTC cron/Kafka event validation, authentication, idempotency and duplicate-event behavior |
| Approval | explicit assignee authorization, single-use decision, reject path and retry-safe reopen |
| Retry | failed-node-only manual retry, stable invocation lineage, attempt ordering and terminal-state guards |
| Events | transactionally coupled history/outbox, after-commit SSE DTO, sequence resume and bounded redacted payload |
| Contracts | strict OpenAPI and v1 inbound/outbound JSON Schemas; MySQL migration, indexes and rollback |
| Frontend | Canvas editing, inspector, validate/publish/run, version restore, triggers, execution timeline and responsive layout |

The deterministic benchmark records 1,000 node transitions and 100 concurrent executions without
external providers. The local gate is transition-pair P99 < 100 ms, all executions terminal-success,
and no throughput regression below 200 executions/s on the recorded development host. This benchmark
does not establish external LLM, Agent, Kafka broker, multi-node failover or production capacity.
