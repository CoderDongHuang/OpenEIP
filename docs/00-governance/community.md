# Community（社区）

> 本文档定义 OpenEIP 的社区治理方式、沟通渠道和参与方式。

---

## 沟通渠道

| 渠道 | 用途 |
|---|---|
| **GitHub Issues** | Bug 报告、Feature Request |
| **GitHub Discussions** | 技术讨论、RFC 讨论、社区交流 |
| **GitHub Pull Requests** | 代码贡献 |
| **GitHub Security Advisory** | 安全漏洞私密报告 |

## 行为准则

所有参与者必须遵守 [CODE_OF_CONDUCT.md](../../CODE_OF_CONDUCT.md)。

核心原则：

- 尊重他人
- 建设性讨论
- 无歧视、无骚扰
- 对事不对人

## 参与方式

### 作为用户

- 下载使用 OpenEIP
- 在 GitHub Discussions 中分享使用经验
- 报告 Bug（通过 Issue）
- 提出功能建议（通过 Issue）
- 参与 RFC 讨论

### 作为贡献者

- 阅读 [CONTRIBUTING.md](../../CONTRIBUTING.md)
- 认领 Good First Issue
- 修复 Bug
- 补充文档
- 补充测试
- 开发新功能

### 成为 Committer

参见 [committers.md](committers.md) 中的晋升机制。

### 成为 Maintainer

参见 [maintainers.md](maintainers.md) 中的晋升机制。

## 决策机制

| 决策类型 | 决策方式 | 说明 |
|---|---|---|
| 日常开发 | PR + Code Review | 由模块相关的 Committer/Maintainer Review |
| 重大功能 | RFC 流程 | 社区讨论 + Maintainer 投票 |
| 架构决策 | ADR 记录 | Maintainer 决策并记录 |
| 治理变更 | Maintainer 投票 | ≥ 2/3 赞成通过 |

## 会议

目前 OpenEIP 不设立定期会议。所有讨论通过异步方式进行（GitHub Discussions、RFC）。

如未来社区壮大，可考虑增加社区会议。
