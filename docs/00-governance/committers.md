# Committers（提交者）

> 本文档定义 OpenEIP 项目的提交者权限范围、职责和提名流程。

---

## Committer 职责

- 在自己负责的模块内提交代码
- 参与相关模块的 Code Review
- 响应自己模块的 Issue
- 参与 RFC 讨论

## Committer 权限

- 在自己负责的模块内创建 Feature Branch
- 提交 PR 并获得优先 Review
- 参与 Architecture Review

## Committer 限制

- Committer **不**能直接 Merge 自己的 PR（必须由 Maintainer Review 后 Merge）
- Committer **不**能直接推送到 main 分支
- Committer 的权限范围限定在自己活跃的模块

## 成为 Committer

### 提名条件

- 至少贡献 5 个被合并的 PR
- 至少参与 3 个 Issue 或 RFC 讨论
- 熟悉 Engineering Principles 和开发流程
- 由至少 1 名 Maintainer 提名

### 提名流程

1. Maintainer 在 GitHub Discussion 发起提名
2. 被提名人提供自我陈述
3. Maintainer 投票（简单多数通过）
4. 投票期不少于 3 天
5. 通过后更新本文档
